# AI 小说生成漫画项目概述

## 1. 项目定位

本项目旨在将小说内容转化为可阅读、可分享的漫画作品。系统负责完成从文本导入、内容处理、AI 分镜、图片生成到漫画展示的完整流程，帮助用户以更低门槛完成漫画创作。

项目核心目标包括：

- 支持多种小说来源接入
- 支持 AI 对文本进行润色和分镜拆解
- 支持基于角色、场景、道具生成漫画图片
- 支持漫画发布、展示、互动与个人管理

## 2. 核心业务流程

项目的主流程可以概括为：

`小说内容输入 -> 文本处理 -> AI 分镜生成 -> 图片生成 -> 审核/重试 -> 漫画展示`

更具体的生成链路如下：

1. 用户导入小说内容。
2. 系统对文本进行整理，必要时执行 AI 润色。
3. 调用 AI 大模型分析剧情，生成每一集所需的角色、场景、道具和分镜描述。
4. 将分镜描述与用户选择的角色设定、场景设定、道具设定组合后，发送给图片生成服务。
5. 系统通过 UUID 或任务 ID 查询图片生成结果。
6. 图片返回前端后进入结果确认流程；生成失败时支持重新生成。
7. 生成完成的内容以漫画形式展示，并进入广场、阅读、评论等后续环节。

## 3. 小说输入与处理

系统支持以下小说导入方式：

- 爬取小说网站链接
- 本地上传文件
- 直接导入 TXT 文本
- 用户手动输入或创作内容

在内容进入漫画生成流程前，系统可提供以下处理能力：

- 文本清洗与结构整理
- AI 润色，提升叙事表达质量
- 长文本拆分，便于按章节或分集处理
- 为后续分镜生成提取基础上下文

其中，整本小说爬取与 AI 润色属于项目亮点能力。

## 4. AI 漫画生成能力

### 4.1 分镜生成

系统先基于小说内容调用 AI 大模型，生成分镜脚本。分镜阶段会输出：

- 角色信息
- 场景信息
- 道具信息
- 镜头描述
- 每一格或每一段画面的提示文本

### 4.2 图片生成模式

项目支持两类生成模式：

- 低配模式：直接基于文字描述生成图片
- 高配模式：基于分镜描述叠加角色图、场景图、道具图等约束进行生成

高配模式更适合追求人物一致性、场景稳定性和漫画连续性的场景。

### 4.3 生成控制

为了保证生成过程可控，系统支持：

- 配置生成图片数量上限
- 查询任务生成状态
- 对失败任务重新生成
- 在图片返回后进行审核或确认

## 5. AI 配置系统

系统提供 AI 配置界面，允许用户根据自身需求调整模型调用参数，同时由开发者提供默认配置。

可配置项包括：

- AI 名称
- AI Key
- AI URL
- Prompt 提示词

该模块的目标是让不同模型服务可灵活接入，降低后续切换模型或扩展服务的成本。

## 6. 用户系统与安全

用户系统负责账号管理与基础安全控制，主要包括：

- 登录与注册
- 密码加密存储
- 双 Token 验证
- 基于 HTTPS 的安全传输

其中，双 Token 验证与 HTTPS 传输保障属于重点安全设计。

## 7. 前端功能模块

### 7.1 项目广场

首页用于展示创作者生成的漫画内容，主要包含：

- 漫画作品展示
- 阅读量排行
- 评论功能

### 7.2 漫画生成模块

该模块是项目核心功能区，负责：

- 根据小说内容生成漫画
- 展示生成流程和生成结果
- 支持重新生成与结果管理

### 7.3 模板管理

模板系统用于复用常见生成风格和参数组合，主要包括：

- 漫画生成模板
- 指定风格图片生成模板
- 与模板绑定的文字提示内容

### 7.4 社区与互动

项目规划中的互动能力包括：

- 论坛板块
- 在线聊天室
- 可选的 AI 对话功能

### 7.5 个人中心 / 我的设置

用户个人设置模块包括：

- 收藏模板
- 收藏漫画
- 修改头像与昵称
- 管理个人账号
- 管理 AI 配置

### 7.6 额外功能

项目还包含以下增强体验功能：

- 主题切换
- 语言切换
- 仪表盘

其中，仪表盘当前仍处于待确定状态。

# 开发规范
## 代码质量与架构

- 深入思考，保持高质量前端实现。
- React 组件保持单一职责，优先复用 `components` 目录已有能力。
- 保持简洁专业，避免重复造轮子，抽象重复逻辑。
- 代码设计需符合低耦合、高内聚原则。
- 函数长度尽量不超过 150 行。
- 单文件不超过 800 行。

## 前后端通信规范

- 前后端通信必须使用 Axios。
- 禁止使用 Fetch API（第三方库内部调用除外）。
